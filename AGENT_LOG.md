## Phase 1 ‚Äî Get It Running
- Local run needs `python3-venv` + `pip` (installed via apt). Created venv, installed core deps.
- **Bug: `requirements.txt` pinned `openai==1.59.0` which does not exist on PyPI** ‚Üí `pip install` failed. `openai` is never imported by the app code. Removed the dead pin.
- **Bug: Dockerfile copied `scripts/*.py` to `/app/scripts/` but CMD runs `python -m uvicorn agent_server:app` from `/app`** ‚Üí module not found in container. Fixed COPY target to `/app/`.
- Ran real API with Redis: `GET /health` ‚Üí `{"status":"ok",...}` confirmed.

## Phase 2 ‚Äî Fix & Harden
- **Real crash bug:** `rate_limiter.py:39` initialized `self._sessions` as a tuple `=()` instead of dict `={}` ‚Üí every `/chat` and `/session` request raised `AttributeError: 'tuple' object has no attribute 'get'` (HTTP 500). Fixed to `= {}`. After fix, `/chat` works (graceful fallback when no LLM configured) and `/health` stable.
- Removed dead import `openai` dependency.
- Extended `.gitignore` (venv, \_\_pycache\_\_, .env.local).
- Secret scan: no committed secrets; `.env` not tracked; `.env.example` uses `change...n` placeholders. OK.
